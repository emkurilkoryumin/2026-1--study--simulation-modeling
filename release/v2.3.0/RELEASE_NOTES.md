# Release v2.3.0

## Общая информация

Релиз `v2.3.0` подготовлен для репозитория
`2026-1--study--simulation-modeling` и отражает актуальное состояние работы
по курсу имитационного моделирования на **18 апреля 2026 года**.
Центральной частью выпуска остаётся лабораторная работа №5, посвящённая
задаче обедающих философов и её исследованию в аппарате сетей Петри
средствами `Julia`.

Новая версия релиза оформлена как полный комплект материалов для хранения,
публикации, передачи преподавателю и последующего воспроизведения. При
оформлении релизных файлов использованы нейтральные технические имена без
персональных названий в стиле `Идрисов` и подобных формулировок.

## Что обновлено в v2.3.0

- добавлен новый отдельный файл защиты
  `simulatio-modeling-lab05-defense.md` для обновлённой презентации `lab05`;
- пересобрано дерево релиза `release/v2.3.0` на основе текущих материалов
  лабораторной работы №5;
- в релиз включены актуальные собранные артефакты отчёта и презентации;
- сохранена подробная структура каталогов `reports`, `presentations`,
  `artifacts` и `archives`, чтобы содержимое релиза было понятно без
  дополнительного просмотра репозитория.

## Что сделано в лабораторной работе №5

- реализована модель задачи обедающих философов в аппарате сетей Петри;
- подготовлены две постановки: классическая сеть и сеть с арбитром;
- выполнен базовый стохастический эксперимент для сравнения двух топологий;
- построены графики эволюции маркировки, сравнительный график состояний
  `Eat_i` и `ODE`-аппроксимация;
- проведено параметрическое исследование по скорости захвата первой вилки
  `left_rate`;
- собраны `CSV` и `TSV`-данные, итоговые графические панели и GIF-анимация;
- подготовлены literate-материалы, производные `.jl`, `.qmd`, `.ipynb`,
  итоговый отчёт, обновлённая презентация, исходный текст выступления и
  новый расширенный файл защиты.

## Что ещё входит в выпуск

- сохранены материалы предыдущих лабораторных работ, уже присутствовавшие в

## Основные результаты лабораторной работы №5

- в классической сети Петри для базового сценария возникает deadlock;
- в сети с арбитром тупик не возникает на всём интервале моделирования;
- при увеличении `left_rate` вероятность взаимной блокировки в классической
  постановке быстро растёт;
- арбитр удерживает вероятность deadlock на нуле во всём исследованном
  диапазоне параметров;
- результаты оформлены в воспроизводимом виде и дополнены отчётом,
  презентацией, графиками, данными и двумя вариантами текста для защиты.
